# DnD Skill — Мастер-план
*Синтез: Клоя + Кремень + Клинок | 2026-03-06 | Обновлён по замечаниям Макса*

---

## TL;DR

Три компонента:
1. **Система записей** — файлы в `workspace/dnd/`, изолировано от памяти Клои
2. **Поиск** — отдельный векторный индекс только по DnD данным (+ отдельный namespace для правил)
3. **Сайт** — Hugo в Docker на pihome, стиль «заметки Торвина», простой HTTP

Приоритет: сначала структура и заметки → саммари → поиск → сайт (позже).

## 2. Изоляция поиска

**Схема: A (путевая) + B (векторный поиск)**

### A — Путевая изоляция (работает сразу)
`memory_search` ищет по `memory/*.md`. DnD в `workspace/dnd/` — нет пересечений.

### B — vectors.db (добавить на этапе 3)
- Хранится прямо в `dnd/vectors.db`
- Embeddings: bge-m3 на `forge.lan:11434`
- **Два namespace в одной базе:**
  - `campaign` — кампания (NPC, квесты, локации, сессии)
  - `rules` — 5e 2024 SRD + хомрулы
- Поиск разделён: «найди в кампании» vs «найди по правилам» — разные скрипты, разные namespace

## 3. SKILL.md — команды

### Перед сессией
- «что было в прошлый раз» / «саммари» → последние 2 сессии + активные квесты + открытые вопросы
- «что делать дальше» → незакрытые квесты + вопросы из last session

### Во время сессии — быстрые заметки
- «запиши: [текст]» → `dnd-note.sh` → `notes/YYYY-MM-DD-live.md`
- «нпс: [имя] — [суть]» → запись с тегом NPC
- «квест: [название]» → запись с тегом QUEST
- «слух: [текст]» → запись в `rumours.md`
- «лут: [предмет]» → запись в `loot.md`

### Поиск по кампании
- «найди в кампании [запрос]» → dnd-search.py (namespace: campaign)
- «кто такой [имя]» → npcs/[slug].md
- «что мы знаем о [локация]» → locations/[slug].md

### Правила
- «правило про [тема]» → dnd-rules-search.py (namespace: rules)
- «спорный момент: [описание]» → поиск + интерпретация со ссылкой на раздел
- Система: **D&D 5e 2024 edition**

### Персонаж и партия
- «мой персонаж» / «Торвин» → characters/torvin-kamneklyatv.md
- «партия» → characters/_party.md
- «инвентарь» / «лут» → loot.md
- «хронология» → timeline.md
- «слухи» → rumours.md

### NPC и локации
- «список NPC» → npcs/_index.md
- «расскажи про [имя]» → npcs/[slug].md
- «про [локацию]» → locations/[slug].md
- «карта [место]» → assets/maps/

### Служебные
- «переиндексируй базу ДнД» → dnd-index.py (перестроить vectors.db)
- «обнови индексы» → dnd-index-rebuild.sh (перестроить все _index.md)

## 5. Relationship graph (линк-граф)

Каждый NPC, квест и локация в frontmatter имеют поле `relations` и `related_*`.
`dnd-index-rebuild.sh` собирает граф из этих полей и выводит в `_graph.md` для читабельности.

Пример запроса: «кто связан с Алистером» → скрипт обходит relations у всех NPC → находит упоминания.

## 7. Сайт «Заметки Торвина»

### Стек
- **Hugo** в Docker
- Docker Compose на pihome рядом с остальными контейнерами
- Простой HTTP, произвольный порт (например 8765)
- Наружу не выставляем (только локалка)
- Публичные заметки — другие игроки могут зайти в локалке

### Дизайн
- Фон: уголь/сланец `#1B1A18`
- Текст: тёплый пергамент `#D9C8A9`
- Акцент: бронза/медь `#A87832`
- Ссылки: `#4E6E6A`
- Текстуры: кожа/бумага 4-8% opacity

### Структура страниц
- `/` — синопсис, последняя сессия, активные квесты, быстрые ссылки
- `/characters/` — партия + листы персонажей
- `/quests/` — активные / выполненные / проваленные
- `/npcs/` — карточки с relation graph
- `/locations/` — локации + встроенные карты
- `/sessions/` — хронология сессий
- `/rumours/` — слухи таверны (из rumours.md, обновляется при build)

### Автодеплой
```
dnd/*.md изменился
  → systemd path unit (inotify)
  → docker compose exec hugo hugo --minify
  → доступно на pihome.local:8765
```
